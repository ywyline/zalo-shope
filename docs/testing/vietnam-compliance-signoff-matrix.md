# 越南专业合规复核与签字矩阵

> 本文件是证据模板，不构成越南法律、税务、会计、医疗器械或商品监管意见。
>
> 正式上线必须由越南本地律师、会计及适用行业合规人员依据届时有效的官方来源签字。

状态：待逐商城外部专业复核与签字；当前空白模板不能关闭任何合规保留项。

## 1. 使用规则

- 美妆和服装商城分别维护；经营主体、数据流、商品范围或供应商不同，不得共用未经说明的结论。
- 状态只允许 `DRAFT`、`UNDER_REVIEW`、`APPROVED`、`REJECTED`、`EXPIRED`、`N/A`。`N/A` 也需要理由、专业复核人和日期。
- 每行必须记录届时有效的越南官方来源、条款/版本和生效日。`REQUIREMENTS.md` 只能作为产品范围，不能作为法律依据。
- 专业人员给出法律/税务/行业结论；内部产品、隐私和安全人员只证明系统控制及证据，不能替代专业意见。
- 资质原件、身份证明、合同和法律意见放在限制访问的合规证据库。仓库只记录脱敏 URI、文件版本和 SHA-256。

## 4. 系统证据映射最低要求

- 公示/政策：发布 URL、三语版本、内容 hash、发布时间、批准人和历史版本。
- 个人数据：数据流图、处理清单、同意/撤回/权利请求用例、保留删除规则、脱敏日志和处理者清单。
- 商品合规：`compliance_requirements` 规则版本、不可变审核记录、提交/审核职责分离、到期门禁、发布快照、强制下架/召回演练。
- 财税：订单、支付、退款、COD、电子发票和对账能力尚未实现时，对应行必须 `BLOCKED` 或附带“上线前不得启用”的明确条件，不能用设计文档代替流水证据。
- 平台安全：双商城越权回归、RBAC、MFA、PII 加密、敏感日志扫描、备份恢复与事件响应演练。
