## 1. 交付结果

- 新增版本化促销生命周期：草稿、不可变发布版本、暂停/恢复、结束、时间窗、类型化品牌/类目/商品/SKU/STORE 目标、互斥与双向叠加声明。
- 新增 `promotion_operations` 商城范围追加写命令表。发布、暂停、结束和优惠券状态命令使用商城+幂等键+请求 hash，重复请求回放首次结果，复用键改请求返回稳定冲突；并发发布只允许一个状态变化和审计记录。
- 新增 `coupons.new_customer_only` 与严格优惠券草稿/状态契约。M3 以同商城零已完成订单基线判断新客，M3 不伪造核销；会员领取仍受商城、状态、时间窗、领取上限和天然幂等约束。
- 可信 VND 计价服务只从服务端 SKU、商品、品牌、类目、默认履约仓库存、促销和优惠券事实重算。计算顺序按 `REQUIREMENTS.md:300` 固定为 `ITEM → COUPON → ORDER`；金额使用整数 VND 与 BigInt 中间计算，整单优惠按当前行应付比例向下取整并以最大余数、SKU 内部 ID 稳定分摊；数量门槛返回独立 `MINIMUM_QUANTITY_NOT_MET`。
- 报价事务使用 PostgreSQL `REPEATABLE READ`，同一事务时间戳和确定性 SHA-256 `quote_hash` 绑定 SKU/商品/品牌/类目/库存/仓库/规则/优惠券事实；响应明确 `order_payable_vnd = null`，只返回包邮资格，不伪造运费或订单金额。
- 新增 `GET /v1/admin/promotions/targets` 同商城候选查询，支持品牌、类目、商品、SKU、编码/三语名称搜索与游标分页；管理端目标编辑器使用候选选择，不要求运营手填 UUID。
- 管理端新增三语促销/优惠券/报价工作台：草稿编辑、目标选择、时间窗、叠加、优惠券绑定、高风险确认、冲突刷新、分页加载、加载/错误/空状态和无副作用管理员预览。共享 VND、状态和目标组件，不复制美妆/服装业务代码。
- 发布版本子表完整性迁移在写入时对旧/新父版本使用 `FOR SHARE`，阻止发布窗口并发写入；优惠券进入非草稿状态后规则字段冻结。E2E 夹具清理遵守审计日志追加写约束，不删除审计历史。

## 2. 安全、隔离与一致性证据

- 管理 API 同时校验管理员令牌、商城授权、`X-Store-Code`、权限 code 和强制 RLS；请求体不接受 `store_id`。目标候选、促销、优惠券、报价事实均按同商城复合关系查询，跨商城目标和令牌/Header 不匹配会被拒绝。
- `store.promotions.manage` 与 `store.promotions.publish` 分离；只读管理员可做无副作用报价预览，券预览明确是假设持券/新客资格，不创建领取、核销、库存或订单事实。
- 发布、暂停、恢复、结束和券状态变化写前后快照审计；`promotion_operations` 与审计日志均保持追加写。API 冲突响应只输出稳定 `details.reason_code`，不回显 SQL、Token、手机号或内部堆栈。
- 报价拒绝客户端价格、折扣、运费、应付金额和 `store_id` 注入；库存只读，不创建预留。零值商品行的最大余数分摊不会收到负向/越界折扣单位。
- 目标类型严格限制为已知目录资源；`CATEGORY` 只匹配当前商品主类目或辅助类目的同一 ID，不递归后代类目。

## 5. 兼容性、限制与未验证项

- 本阶段不实现购物车、地址、结算、订单、COD、支付、物流、退款或售后；报价不是最终订单金额，M4 必须重新加载事实、锁库存、验证优惠券并保存快照。
- Web E2E 不能替代 Zalo Mini App 宿主真机验收；真实 Zalo 凭据、Checkout、支付/物流沙箱、生产对象存储/CDN、远程 CI 和越南专业合规复核仍未验证，也未使用生产凭据或执行外部发布/部署。
- 当前 Windows 环境把 2984–3083 列为排除端口；Playwright 支持 `E2E_API_PORT`，标准测试环境从 `.env.test.example` 使用 3100。遇到 `listen EACCES` 时应显式选择未被排除的端口。Vite 通过同源 `/api` 代理到该端口，避免跨源 CORS。
- 测试清理保留 append-only 审计历史；测试账号、商品和促销事实仍按隔离夹具清理，不改变生产不可变约束。
