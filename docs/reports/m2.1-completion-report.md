# M2.1 契约冻结完成报告

日期：2026-07-17

结论：M2.1 已完成。字段字典、状态、权限和 OpenAPI 草案已经冻结，类目树、SKU 规格组合、三语完整度和合规上架门禁已由可执行单元测试固定。按阶段计划，本阶段未创建数据库迁移、API 业务实现或页面；M2.2 只能在这些已冻结契约上实施，如需改变关键不变量必须先更新计划和本报告。

## 已交付

- `docs/database/m2-data-dictionary.md`：定义品牌、类目、属性模板版本、SPU、SKU、媒体、合规、商品版本和页面装修的数据结构、索引、复合外键、RLS、不可变规则及回滚边界。
- `docs/security/m2-permission-matrix.md`：定义商城目录、发布、合规审核和装修权限，包含审核人/提交人分离、商城作用域和拒绝测试要求。
- `docs/api/openapi.m2.yaml`：定义买家目录读取和后台目录管理的增量契约；请求体不接收 `store_id`，买家 SKU 响应不包含成本价。
- `packages/domain/src/catalog.ts`：实现类目位置、SKU 组合键、翻译完整度和商品发布决策的纯领域规则。
- `packages/contracts/src/catalog.ts`：实现稳定业务编码、三语内容、商品草稿和整数 VND SKU 输入的严格 DTO 校验。
- `docs/plans/m2-implementation-plan.md` 与总计划已记录用户批准、M1 真机风险结转和 M2.1 完成状态。

## 验证证据

- 测试驱动记录：新增测试最初因目录领域模块和 DTO schema 不存在而失败；实现后纳入全仓测试。
- `corepack pnpm verify`：格式、Lint、类型检查、46 个单元测试、全部生产构建和 Prisma schema 校验通过。
- `corepack pnpm test:integration`：4 个测试文件、18 个既有数据库/API/基础设施回归测试通过。
- OpenAPI YAML 由 Prettier 成功解析并格式化。
- 项目 Docker 容器已在集成测试后执行 `infra:down` 清理。

## 风险与限制

- M2 数据表、迁移、RLS、种子和真实 API 尚未实现，这是 M2.2 及后续子阶段的范围，不得将本报告解读为 M2 全部完成。
- OpenAPI 当前是冻结草案；M2.4 实现时仍需加入契约测试并验证错误码、分页、幂等和商城上下文绑定。
- 真实对象存储上传、后台认证后 UI、Zalo Mini App 项目和真机验收尚未执行。
- 用户已接受 Zalo Mini App 真机验收作为结转风险；该决定只解除 M2 启动门槛，不代表风险关闭或验收通过。

## 下一阶段门槛

M2.2 先编写迁移/隔离失败测试，再实现 Prisma schema、SQL 迁移、复合外键、RLS、审计和本地测试种子；必须验证空库迁移、重复 deploy、跨商城拒绝、向前修复路径和仅限无业务数据环境的人工回滚。
