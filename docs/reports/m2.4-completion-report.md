# M2.4 商品、SKU、媒体与合规后台 API 完成报告

日期：2026-07-18

结论：M2.4 已完成。商品草稿、SKU、媒体、合规审核和发布门禁已接入 M1 商城上下文、RBAC 与审计链路；M2.5 装修后台 API 与管理端 UI 尚未开始。

## 已交付

- 新增商城范围商品列表/创建、SKU 草稿集合替换、商品媒体关联、提交审核和发布命令；请求体不接受 `store_id`，高风险写命令使用 `expected_version`。
- 商品只能绑定同商城启用品牌、启用末级类目及该类目的已激活主属性模板；主类目不能重复为辅助类目。
- SKU 规格必须来自商品固定模板，必填规格不可缺失，组合 key/hash 规范化且商城内编码、商品内组合继续由数据库唯一约束保护；VND 价格只接受安全非负整数。
- 新增 S3 兼容媒体端口与 5 分钟 checksum 绑定 PUT URL；对象键包含环境、商城和资源，确认阶段核对长度、MIME 和可用 checksum。SVG 在清洗器交付前隔离，不会伪装 READY。
- 本地 Compose 新增幂等 `minio-init`，创建配置的 local/test bucket；真实 MinIO 上传、错误 checksum 拒绝、HEAD 核验和清理均有集成测试。
- 新增合规资料提交与审核命令；提交需要 `store.catalog.manage` 和 `store.compliance.read`，审核需要独立 `store.compliance.review`，并禁止提交人审核自己的记录。
- 提交与发布均重新执行品牌、类目、越南语、主图、SKU、必填属性和合规门禁；发布生成带内容 hash 的不可变 `product_versions` 快照。
- 审计 JSON 现在正确保留 ISO 时间，并把安全范围 BigInt VND 转为 JSON 整数，避免审计或 HTTP 响应序列化失败。

## 契约与兼容性

- 向 `/v1` 兼容新增商品媒体关联、合规提交、媒体确认路径，并为提交/发布增加显式 `expected_version` 请求体。
- `docs/api/openapi.m2.yaml`、M2 数据字典和实施计划已同步；无 Prisma schema 或生产环境变量变化。
- 新增 AWS 官方 S3 client/presigner 依赖，固定版本 `3.1088.0`；许可证与现有 AWS SDK 依赖一致。
- 应用代码可回退；已发布商品版本、已审核合规记录和已确认媒体继续按现有向前修复原则保留。

## 自动化证据

- `corepack pnpm test:unit`：49/49 通过，包括媒体元数据、合规输入和审计 BigInt/日期 JSON 安全回归。
- `corepack pnpm test:integration`：7 个文件、34/34 场景通过，包括真实 local MinIO checksum 上传、M1/M2 数据隔离、商品/SKU、跨商城媒体拒绝、合规禁止自审及不可变发布快照。
- M2 数据库不可变性测试夹具改用 `blocking=false` 的测试合规要求，避免重复测试残留的全局要求污染后续上架门禁；不改变生产规则。

## 安全与风险验证

- 普通请求通过令牌、`X-Store-Code` 和 `store_id` 三方一致性确定商城；跨商城媒体 ID 返回不泄漏存在性的 404。
- 预签名上传绑定 checksum，错误内容由 MinIO 拒绝；未确认、失败、隔离或错误资源类型媒体不能关联商品/合规资料。
- 成本价受现有日志敏感字段处理保护；商品接口仍不向买家开放，本阶段没有买家价格响应。
- 合规最终规则仍需越南专业人员复核；当前实现只执行可配置要求和审批分权，不构成法律判断。

## 已知限制与下一阶段

- SVG 清洗、病毒扫描、图片尺寸探测/转码/CDN 属于后续媒体处理任务；当前安全策略是隔离而非假成功。
- 本阶段没有商品属性值编辑 API、批量逐项命令、商品导入、管理端 UI 或买家目录 UI；分别在后续 M2 子阶段补齐。
- 生产对象存储 bucket、域名、CDN 和密钥管理仍未确定；只验证了 local/test MinIO。
- Zalo Mini App 真机验收继续作为用户已接受的结转风险。
- 下一阶段：M2.5 装修后台 API 与管理端 UI。
