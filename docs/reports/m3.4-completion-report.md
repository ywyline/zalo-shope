# M3.4 三语搜索、筛选、历史与买家端搜索体验完成报告

> 状态：已完成
>
> 日期：2026-07-20
>
> 范围：`docs/plans/m3-implementation-plan.md` 的 M3.4；不包含 M3.5 促销/优惠券/价格引擎、M3.6 购物车或 M4 订单。

## 1. 交付结果

- 新增三语商品搜索投影服务：每个可展示商品生成 vi/zh/en 文档，缺失语言回退越南语；文档包含商品、品牌、主/辅助类目、父类目、可筛选商品/SKU 属性和最低在售 SKU 价格。
- 商品发布/停用以及品牌、类目内容或状态变更在原商城事务内同步或移除搜索文档；父类目变更会覆盖使用其子类目的商品。查询侧再次校验商品、品牌和主类目当前可见状态，避免陈旧派生文档造成泄漏。
- 新增逐商城 `search:rebuild` 运维命令，使用 runtime RLS 上下文在单一事务内重建派生文档并写入前后计数审计；商品、库存、会员历史和热门词事实不受重建影响。
- 新增 `GET /v1/search/products`、`GET /v1/search/suggestions`、`GET /v1/search/facets` 与会员搜索历史读取/清空实现。搜索支持越南语有/无变音符号、`đ/Đ`、中文、英文、FTS/trigram、品牌、一级/二级类目、行业属性、价格、有货筛选以及相关度/上新/价格稳定游标排序。
- 新增公开搜索 Redis 固定窗口限流；来源地址仅以 HMAC 摘要进入短期键。匿名搜索不保存标识；明显手机号、Token、密码或长随机串样式的查询不写会员历史或商城聚合。
- 新增共享内存会员会话，身份令牌仍不进入 LocalStorage/SessionStorage；已认证会员按商城保存最近 50 条去重历史并可清空，未认证用户仍可搜索但不产生个人历史。
- 新增美妆/服装共用的移动端搜索页：250 ms 联想、历史词、排序、底部筛选、价格/品牌/类目/行业属性/库存、分页加载、图片失败占位、加载/错误/重试/空状态和同商城真实无结果推荐。界面文案完整覆盖 vi/zh/en。
- 为真实筛选 UI 向 OpenAPI 增加向后兼容的只读 `/v1/search/facets` 契约；未删除或重命名 M1-M3.3 接口。本阶段复用 M3.2 schema，没有新增迁移或改写已激活属性模板。

## 2. 安全、隔离与一致性证据

- 公共 API 先从受信 `X-Store-Code` 解析活动商城，再在强制 RLS 事务中执行；会员令牌商城必须与 Header 一致。测试使用另一商城哨兵验证结果、筛选项、联想和历史均不跨商城。
- 所有搜索 SQL 使用 Prisma 参数片段；严格 DTO 限制查询长度、重复筛选、金额、游标和未知字段。恶意 SQL 形态查询只作为参数执行，不改变查询结构或读取另一商城。
- 历史按 `(store_id, member_id, locale, folded_query)` 去重并裁剪；热门词按商城和语言聚合。Redis 仅保存短期限流计数，不承担搜索、库存或会员事实。
- 有货状态从当前商城启用默认履约仓的库存余额实时聚合，不写入搜索投影；价格使用投影中的整数 VND 并在 API 边界验证 JavaScript 安全整数。
- E2E 搜索夹具通过正式库存调整原语创建非零库存，并使用独立的先草稿后激活、可筛选行业模板；未绕过属性模板不可变规则。相关集成夹具按搜索文档、库存余额、SKU 和商品的依赖顺序清理，重复全量测试后孤儿余额与孤儿检索文档均为 0。

## 4. 兼容、回滚与限制

- 本阶段复用 M3.2 数据表，没有新增数据库迁移。应用回滚时停止搜索路由和同步调用但保留派生文档；如投影不一致，按商城执行重建，不回滚或删除商品、库存、历史与审计事实。
- `on_promotion` 和 `promotion_summary` 保留冻结契约，但 M3.5 尚未产生可由价格引擎确认的活动促销事实；当前不伪造促销，`promotion_summary` 为 `null`，`on_promotion=true` 返回空结果，Mini App 暂不暴露促销筛选控件。M3.5 接入真实发布版本与价格规则后启用。
- PostgreSQL `simple` FTS 配合 trigram 满足当前 P0 规模和三语回归；尚未完成生产数据规模的查询计划/压测。达到已批准阈值后再评估索引调整或外部搜索服务，不在本阶段提前引入。
- Zalo 宿主真机、生产对象存储/CDN、生产凭据和越南专业合规复核仍是上线前置条件；Chromium/WebKit Web E2E 不替代真机验收。
