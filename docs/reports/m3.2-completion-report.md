# M3.2 数据库迁移、RLS、初始回填与基础种子完成报告

> 状态：已完成
>
> 日期：2026-07-20
>
> 范围：仅 M3.2；未进入 M3.3 库存服务/API、M3.4 持续搜索同步或任何 M4 能力

## 1. 完成内容

- 在 Prisma schema 中加入 11 个冻结枚举和 18 张商城业务表，覆盖仓库/库存、搜索投影、促销/优惠券和购物车。
- 所有 M3 表均使用非空 `store_id`；商城内关系使用复合外键与 `UNIQUE (store_id, id)`，并启用、强制 RLS 及 `store_id` 不可变触发器。
- 库存余额使用数据库生成的 `available = on_hand - reserved`；数量、版本、差量、前后余额和动作类型由检查约束保护。
- 库存操作与流水为追加写事实；预留终态不可逆。促销/优惠券状态转换、发布版本、越南语文案、目标及优惠利益字段受数据库约束和触发器保护。
- 搜索安装 `unaccent`、`pg_trgm`，创建 FTS、trigram、类目和筛选 GIN 索引；`search_vector` 为数据库生成列。
- 迁移会幂等回填 M2 已存在的当前发布商品，保存原文、小写文本、去越南语变音文本、价格、筛选投影与发布版本；不创建库存、销量、订单、物流或核销假事实。
- 生产迁移只增加六个权限目录项，不修改既有角色授权。local/test 种子显式为 `store-admin` 授权，并为两个测试商城创建三语默认测试仓；已有 SKU 仅补零余额。
- 提供带事实数据保护门禁的 local/test `down.sql`；生产和已有真实 M3 数据环境继续采用向前修复。

## 3. 迁移演练证据

| 场景                              | 结果                                                                                                         |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| 全新空库连续两次 `migrate:deploy` | 首次应用 5 个迁移，第二次无待执行迁移                                                                        |
| local/test 连续两次 seed          | 成功；2 个默认测试仓，非零余额为 0，6 个 M3 权限均存在                                                       |
| M2 已有发布商品升级               | 成功回填 1 个越南语文档；`Son dưỡng ĐẸP!` 生成 `son duong dep ...`，价格 99000 VND、来源版本 3、FTS 向量非空 |
| 生产既有角色权限                  | 6 个目录项存在，M3 隐式授权数为 0，原权限保持不变                                                            |
| 扩展缺失故障注入                  | 在创建业务对象前以明确的 `M3.2 requires ... extension` 错误失败                                              |
| local/test 无真实 M3 数据 down    | 成功；删除迁移记录后可重新前滚并再次 seed                                                                    |
| 最终空库结构抽查                  | `search_vector` 为 stored generated 列；抽查 5 张表全部启用并强制 RLS                                        |

升级演练最初发现 Prisma 未把 `Unsupported("tsvector") @default(dbgenerated())` 输出为生成列，导致已有商品回填违反非空约束。迁移已改为显式 PostgreSQL stored generated column，并在 M2 已有数据和最终空库场景重新验证。
