# M2.8.3 受限 XLSX 商品导入与导出完成报告

> 日期：2026-07-19
>
> 结论：受限 XLSX 模板、商品草稿导入、逐行报告、商品导出和三语管理端入口已按批准边界完成；自动化与真实浏览器门禁通过。M2.8.4 可重复浏览器 E2E 仍未完成，不自动进入 M3。

## 1. 完成范围

- 新增 XLSX 模板下载、dry-run/正式商品草稿导入和当前授权商城的商品导出；CSV 与 XLSX 共用同一组 20 列契约、行级领域转换及数据库写入服务。
- 导入只创建 `DRAFT` 商品/SKU，不接受商城归属、商品状态、库存、媒体或合规数据；正式导入继续使用逐商品事务和现有唯一约束。
- 导出固定为一行一个 SKU，无 SKU 商品保留空 SKU 行；包含三语内容、业务编码、类目、售价和规格选项，不包含成本价、内部 ID、商城 ID、合规凭据或对象存储信息。
- 管理端新增模板下载、商品导出、单文件选择、默认 dry-run、正式导入 `IMPORT` 二次确认、逐行结果和加载/错误反馈；越南语、中文、英文文案完整。
- 本阶段不新增数据库迁移，不实现库存导入、异步大文件导出或 M3 业务。

## 2. API、权限与兼容性

| 接口                                                               | 权限                   | 结果                           |
| ------------------------------------------------------------------ | ---------------------- | ------------------------------ |
| `GET /v1/admin/catalog/products/imports/template.xlsx`             | `store.catalog.read`   | 下载固定 `products` 工作表模板 |
| `POST /v1/admin/catalog/products/imports/xlsx?dry_run=true\|false` | `store.catalog.manage` | 返回与 CSV 一致的逐商品结果    |
| `GET /v1/admin/catalog/products/exports/products.xlsx`             | `store.catalog.read`   | 导出当前授权商城商品           |

- 现有 CSV 路由和响应保持兼容；XLSX 解析结果进入同一导入服务，没有复制第二套写入逻辑。
- 所有读写从认证后的商城上下文派生，服务端不信任文件中的商城信息；导出查询仍在 RLS 事务内完成。
- 同步导出最多 10,000 行且响应最大 16 MiB，超限显式失败，不静默截断。

## 3. 文件安全边界

- 上传上限 2 MiB、最多 128 个 ZIP 条目、单条目解压后最多 8 MiB、总声明解压大小最多 16 MiB、压缩比最多 100:1，导入数据最多 2,000 行。
- 在第三方解析器前检查 ZIP 中央目录并拒绝 ZIP64、分卷、加密、重复/危险路径、未知压缩方法、宏、VBA、对象、图片、图表、外部链接、DTD/实体和错误单元格。
- 只接受一个可见且命名为 `products` 的工作表；严格要求 20 列，拒绝公式、额外/隐藏工作表、非安全整数和超长文本。
- 导出文本强制按字符串写入；以 `= + - @` 开头的值增加前导单引号，避免表格公式注入。
- 新增 `read-excel-file@9.3.2` 与 `write-excel-file@4.1.1` 固定版本；两者为 MIT、要求 Node.js 18+，与项目 Node.js 24 基线兼容。生产依赖审计无已知漏洞。

## 6. 已知限制与下一步

- 库存导入与库存调整仍属于 M3，不能通过商品 XLSX 导入越界实现。
- 同步商品导出超过 10,000 行时拒绝请求；若后续确有大规模需求，应另行设计带商城隔离、审计和对象存储生命周期的异步任务。
- M2.8.4 仍需建立可重复浏览器 E2E，持续覆盖双商城、三语、桌面/移动视口与关键管理端交互。
- Zalo 真机、生产对象存储/CDN 和越南专业合规复核仍是外部上线前置条件，不因本阶段完成而关闭。
