# M2.5 装修后台 API 与管理端 UI 完成报告

> 日期：2026-07-18
>
> 结论：实现与自动化门禁通过；真实浏览器验收通过，浏览器 E2E 尚未固化为仓库脚本。

## 1. 交付范围

- 新增页面列表、创建草稿、完整替换草稿和发布 API；所有请求从可信请求上下文绑定商城，并执行 `store.content.read/manage` RBAC。
- 页面聚合使用 `expected_version` 乐观锁；已发布版本由数据库触发器保持不可变，再次编辑创建下一草稿版本。
- 模块支持六种类型、启停、唯一排序、展示时间窗、三语标题/摘要/按钮、结构化内容配置、背景配置、媒体和内部/外部跳转。
- 发布重新校验三语、时间窗、同商城可用目标、READY 页面媒体和外部 HTTPS 主机白名单；要求输入页面编码二次确认。
- 创建、草稿替换和发布写入商城范围审计；页面媒体上传/确认改为按资源选择 `store.content.manage` 或 `store.catalog.manage`。
- 管理端新增三语内容工作台：页面切换/创建、模块新增/删除、拖放及键盘排序、时间窗、启停、跳转、主题实时预览、发布确认，以及加载、空、错误、无权、冲突和重试状态。
- 商城切换通过组件重新挂载立即清除上一商城页面状态；美妆和服装预览共用组件并由商城主题分支表现圆角/视觉差异。

## 2. 安全与隔离证据

- API 集成测试覆盖只读角色拒绝写入、请求商城代码与 ID 不匹配、跨商城类目目标伪造、过期版本冲突和审计事件。
- 页面媒体 ID 必须属于当前商城、状态为 `READY`，且对象键包含当前 `store_id/page` 资源路径。
- 外部目标默认全部拒绝；仅精确命中 `CONTENT_EXTERNAL_TARGET_HOSTS` 的 HTTPS 主机可保存和发布。
- DTO 严格拒绝请求体中的额外商城字段、重复排序、缺失语言、不安全 URL、无目标类型的目标值和任意内容配置字段。

## 4. 兼容与回滚

- 本阶段不新增数据库迁移，复用 M2.2 已部署页面关系和不可变触发器；API 仅向 `/v1/admin/content` 增加端点。
- 应用可回滚到前一版本；已发布页面版本和审计记录保留。外部跳转可通过清空 `CONTENT_EXTERNAL_TARGET_HOSTS` 立即关闭。

## 5. 已知限制与后续事项

- 浏览器验收已真实执行，但仓库仍没有可重复运行的浏览器 E2E 脚本；本报告不以手工浏览器证据替代未来自动化项目。
- M2.5 不开放买家端页面读取接口，也不把管理端预览冒充线上首页；真实买家首页、分类、品牌馆、列表、详情和移动端主题链路属于 M2.6。
- 生产外跳白名单、CDN/对象存储域名和 Zalo 真机环境仍需在部署配置与生产验收阶段明确。
