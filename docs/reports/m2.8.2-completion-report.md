# M2.8.2 目录与合规管理端工作台完成报告

> 日期：2026-07-19
>
> 结论：M2.8.2 已完成并通过自动化与真实浏览器门禁；只关闭目录/合规 PC 管理端保留，不代表 XLSX/导出、可重复浏览器 E2E、Zalo 真机或生产媒体链路完成，也不自动进入 M2.8.3 或 M3。

## 1. 交付结果

- 新增品牌、两级类目、属性模板、商品和合规五区管理端工作台，复用一套代码和商城主题/配置；商城切换通过 `store.id` 重新挂载，旧商城选中项、表单与通知不跨店保留。
- 品牌支持越南语、中文、英文创建/编辑、推荐、排序、状态与官网；类目支持三语、两级树、排序、状态和启用主属性模板绑定。
- 属性模板支持定义构建、有限选项、版本查看和业务编码二次确认启用；非法用途/类型组合仍由共享 Zod 契约和服务端复核。
- 商品支持三语草稿创建、安全 SKU 摘要、M2.8.1 类型化非规格属性编辑、发布门禁问题、提交、确认发布与确认停用；不暴露或替换 SKU 成本价。
- 合规区展示启用要求、脱敏编号、日期、状态、媒体数量和商品摘要，并支持带决策词与意见的批准/驳回；提交人/审核人分离仍由服务端可信执行。
- 新增 `GET /v1/admin/compliance/overview` 严格查询契约和安全读模型；类目列表向后兼容增加同商城模板绑定投影。没有数据库迁移。

## 2. 安全、隔离与兼容

- 合规概览要求 `store.compliance.read`，最多返回 100 条，可按商品 UUID 和状态过滤；响应不包含原始证件号、媒体对象键、下载地址、审核资料正文、提交人或审核人标识。
- 所有读写继续携带认证商城上下文、`store_id` 条件并运行在强制 RLS 事务内；集成测试确认跨商城商品过滤不泄露存在性，类目绑定只投影同商城关系。
- 编辑和状态命令携带服务端版本；模板启用、商品发布/停用与合规审核使用明确二次确认，不以前端判断替代 RBAC、状态机或审核人分离。
- API 仅向 `/v1` 增加路由/字段；应用回滚不改变已保存目录、属性、审核或审计数据。生产对象存储与媒体二进制管理没有假成功实现。
