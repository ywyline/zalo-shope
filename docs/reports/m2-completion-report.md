## 2. 已达到的 M2 能力

- 一套代码承载美妆与服装商城，品牌、类目、模板、商品、SKU、媒体、合规、版本和页面数据均以强制 RLS 与复合关系隔离。
- 商品发布门禁覆盖越南语、启用品牌/末级类目、主图、SKU、必填属性和合规记录；发布生成不可变、可追溯快照。
- 首页装修具备草稿/发布版本、三语、时间窗、排序、媒体、同商城跳转和 HTTPS 外跳允许列表；管理端内容工作台通过真实浏览器验收。
- 买家端首页、分类、品牌馆、商品列表/详情和 SKU 选择使用真实公开 API，同一组件树呈现双商城主题、三语回退和 M3 前不可购买状态。
- 商品 CSV 草稿导入、版本历史、批量停用/移动已接入现有 RBAC、审计、乐观锁和整数 VND 原则；没有越界创建库存或交易假实现。

## 3. 当前总体验证基线

- 最新 `corepack pnpm verify` 通过：格式、lint、类型、63 项单元测试、全部生产构建和 Prisma schema 校验。
- 最新 `corepack pnpm test:integration` 通过：10 个文件、47 项，覆盖基础设施、身份、RBAC、RLS、跨商城关系、目录、商品、媒体、合规、内容、买家 API、导入、版本和批量操作。
- 生产依赖审计无已知 high 级漏洞；Compose 配置、`git diff --check` 与本轮敏感信息扫描通过。
- M2.5/M2.6 已分别完成管理端内容工作台和买家目录真实浏览器验收；M2.7 无 UI 变化，没有把构建检查冒充新的浏览器验收。

## 4. 验收保留与影响

| 保留项                              | 影响                                                                | 处理建议                                          |
| ----------------------------------- | ------------------------------------------------------------------- | ------------------------------------------------- |
| 商品属性值编辑 API 未交付           | 复杂行业模板中的非规格必填属性无法由运营完整维护，可能阻塞上架      | 进入 M3 前先形成 M2 收尾变更或由用户明确接受      |
| 品牌/类目/商品/合规管理端 UI 未交付 | 后台只有内容装修 UI；目录运营仍只能调用 API，不满足完整 PC 后台体验 | 独立补齐三语桌面/窄屏工作台并做浏览器验收         |
| XLSX 与商品导出未交付               | 当前只支持 UTF-8 CSV 导入，与需求中的 Excel 运营习惯仍有差距        | 重新评估维护良好的 XLSX 依赖和解压资源限制后实现  |
| 自动化浏览器 E2E 项目缺失           | 已有真实浏览器证据不能持续回归                                      | 建立可重复 E2E 脚本后纳入 `verify` 或独立 CI 门禁 |
| Zalo 真机与生产对象存储/CDN 未验收  | 不能宣称真实宿主身份、生产媒体链路和设备体验完成                    | 等待项目/测试成员/provider/云配置后补验           |
| 越南行业合规规则未由专业人员复核    | 现有可配置门禁不构成法律确认                                        | 上线前由越南法律、税务和行业人员审核              |

因此，M2 可记录为“各子阶段已实施、自动化基线通过”，但不满足 `AGENTS.md` 对完整产品/UI 验收的无保留完成定义。进入 M3 前应由用户选择先补齐上述 M2 产品缺口，或明确接受其作为结转风险；当前批准不允许自动推进 M3。
