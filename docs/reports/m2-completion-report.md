## 2. 已达到的 M2 能力

- 一套代码承载美妆与服装商城，品牌、类目、模板、商品、SKU、媒体、合规、版本和页面数据均以强制 RLS 与复合关系隔离。
- 商品发布门禁覆盖越南语、启用品牌/末级类目、主图、SKU、必填属性和合规记录；发布生成不可变、可追溯快照。
- 首页装修具备草稿/发布版本、三语、时间窗、排序、媒体、同商城跳转和 HTTPS 外跳允许列表；管理端内容工作台通过真实浏览器验收。
- 买家端首页、分类、品牌馆、商品列表/详情和 SKU 选择使用真实公开 API，同一组件树呈现双商城主题、三语回退和 M3 前不可购买状态。
- 商品 CSV 草稿导入、版本历史、批量停用/移动已接入现有 RBAC、审计、乐观锁和整数 VND 原则；没有越界创建库存或交易假实现。
- 商品运营可读取固定模板并以类型安全的全量命令维护非规格属性；商城隔离、草稿状态、乐观锁、停用选项、多值规则、前后快照审计和必填文本越南语门禁均已自动化验证。
- PC 管理端已提供品牌、两级类目、主模板绑定、模板版本、商品草稿、类型化属性、发布门禁与合规审批五区工作台；高风险操作使用显式确认，合规概览不返回原始编号、对象键、下载地址或操作人标识。

## 3. 当前总体验证基线

- 最新 `corepack pnpm verify` 通过：格式、lint、类型、66 项单元测试、全部生产构建和 Prisma schema 校验。
- 最新 `corepack pnpm test:integration` 通过：12 个文件、54 项，覆盖基础设施、身份、RBAC、RLS、跨商城关系、目录、商品属性、脱敏合规概览、媒体、内容、买家 API、导入、版本和批量操作。
- 生产依赖审计无已知 high 级漏洞；Compose 配置、`git diff --check` 与本轮敏感信息扫描通过。
- M2.5/M2.6 已分别完成管理端内容工作台和买家目录真实浏览器验收；M2.8.2 完成目录合规工作台真实 API、双商城隔离、三语、1440×900、900×900 与 390×844 验收。

## 4. 验收保留与影响

| 保留项                             | 影响                                                         | 处理建议                                          |
| ---------------------------------- | ------------------------------------------------------------ | ------------------------------------------------- |
| XLSX 与商品导出未交付              | 当前只支持 UTF-8 CSV 导入，与需求中的 Excel 运营习惯仍有差距 | 重新评估维护良好的 XLSX 依赖和解压资源限制后实现  |
| 自动化浏览器 E2E 项目缺失          | 已有真实浏览器证据不能持续回归                               | 建立可重复 E2E 脚本后纳入 `verify` 或独立 CI 门禁 |
| Zalo 真机与生产对象存储/CDN 未验收 | 不能宣称真实宿主身份、生产媒体链路和设备体验完成             | 等待项目/测试成员/provider/云配置后补验           |
| 越南行业合规规则未由专业人员复核   | 现有可配置门禁不构成法律确认                                 | 上线前由越南法律、税务和行业人员审核              |

商品属性值编辑 API 和目录/合规管理端 UI 保留已分别由 M2.8.1、M2.8.2 关闭；证据见对应完成报告。其余保留不变。因此，M2 可记录为“已实施子阶段门禁通过”，但仍不满足 `AGENTS.md` 对完整产品/UI 验收的无保留完成定义。进入 M3 前应继续补齐上述 M2 产品缺口，或由用户明确接受其作为结转风险；当前批准不允许自动推进 M2.8.3 或 M3。
