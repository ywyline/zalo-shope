# M2.7 商品导入、版本与批量操作完成报告

> 日期：2026-07-19
>
> 结论：M2.7 已批准范围中的 CSV 导入、版本读取、批量停用/移动、契约与自动化门禁已完成；XLSX 按依赖门禁未引入。M2 总体验收仍有保留，详见 `docs/reports/m2-completion-report.md`。

## 1. 已交付范围

- 新增带 UTF-8 BOM 的固定 CSV 模板下载，以及 `dry_run`/实际导入接口。文件限制为 2 MiB、2,000 个数据行、每商品 500 个 SKU；严格拒绝非法 UTF-8、NUL、畸形引号、未知/乱序表头、超限列和不安全字段。
- CSV 使用业务编码引用品牌、主/辅助类目和规格选项，不接受 `store_id`、库存、状态、媒体、合规审批或发布字段。业务编码统一规范化，三语内容正确支持越南语变音符号。
- 同商品多 SKU 使用多行表达，商品字段必须组内一致；每个商品及全部 SKU 使用单一商城 RLS 事务，不同商品允许部分成功，响应逐行返回 `VALIDATED`、`IMPORTED`、`FAILED` 与稳定错误码。
- 导入只创建 `DRAFT` 商品与启用 SKU，不自动提交审核或发布，不伪造媒体、合规或库存结果；整数 VND 继续由共享 Zod 契约和数据库 `bigint` 保护。
- 新增商品发布版本列表和详情。查询同时约束可信商城上下文与 `store_id`；数据库快照保持不可变和完整，`store.catalog.read` 响应递归移除成本价，关闭历史版本旁路泄漏。
- 新增最多 100 项、要求二次确认码和 `expected_version` 的批量停用与主类目移动。每项独立事务、独立审计和部分失败；停用保留历史版本，移动只允许 `DRAFT/UNPUBLISHED`，已有 SKU/属性时禁止跨属性模板移动。
- 同步专项计划、OpenAPI 0.2.7、数据字典、权限矩阵与 M2 总结；无数据库迁移、环境变量或生产依赖变化。

## 2. 安全、隔离与一致性证据

- 模板和导入分别要求 `store.catalog.read/manage`，批量停用要求 `store.catalog.publish`，批量移动要求 `store.catalog.manage`；未认证模板读取被拒绝。
- API 集成测试为同一管理员显式授权双商城后，以服装商城上下文读取美妆商品版本仍返回 404，证明不是依赖界面过滤或角色缺失造成的假隔离。
- dry-run 对真实 PostgreSQL 零写入；实际导入中缺失品牌的商品逐行失败，同文件合法商品成功且生成单个草稿/SKU，不存在组内部分数据。
- 批量命令中合法项成功、未知商品失败；成功项版本递增，失败项不影响其他事务。重复商品目标在任何写入前由严格契约拒绝。
- 集成测试直接尝试修改 `PUBLISHED` 版本快照并被数据库触发器拒绝；版本详情回归确认嵌套 `costPriceVnd` 不进入响应。
- 导入、移动、停用分别生成 `catalog.product.imported`、`catalog.product.main_category_moved`、`catalog.product.disabled` 商城范围审计事件。

## 4. 兼容、迁移与回滚

- 本阶段不新增或修改 Prisma schema、迁移、种子和外部对象；所有接口是 `/v1` 向后兼容新增。
- 应用可回滚到 M2.6；已导入草稿、SKU 与审计记录保留，可由受权运营停用。误停用没有未审计恢复捷径，需以后续受审向前修复命令处理。
- 批量停用不修改历史发布版本；主类目移动在属性模板不兼容时拒绝，不清空或重写已有 SKU/属性。
- CSV 文件仅在请求内存中受限解析，不写对象存储，无孤立上传对象清理负担。

## 5. 未执行项与已知限制

- 本阶段没有 UI 改动，因此未重复执行浏览器交互验收；M2.6 已有双商城、三语、390×844/1280×900 买家目录浏览器证据。本仓库仍没有自动化浏览器 E2E 脚本。
- 本阶段不涉及媒体上传变化；local/test MinIO 由全量基础设施/既有集成测试覆盖，生产对象存储、CDN、扫描和派生图片仍未验收。
- 真实 Zalo Mini App 项目、测试成员、真机和生产 provider 配置仍缺失，继续作为用户已接受的结转风险。
- 未新增 XLSX 依赖。主流 `exceljs` 稳定版维护节奏不满足本仓库新依赖门禁，新作用域发布链采用度不足；按已批准计划先交付 CSV。XLSX 仍是明确未实现格式。
- M2.4 结转的商品属性值编辑 API、品牌/类目/商品管理端 UI，以及商品导出尚未交付；这些项目使 M2 总体验收保留，不能以本阶段 API 完成代替。
- 不实现库存导入、搜索、购物车、促销或计价；库存仍严格属于 M3，且本轮未获得自动进入 M3 的授权。
