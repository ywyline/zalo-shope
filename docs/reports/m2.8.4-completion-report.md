# M2.8.4 可重复浏览器 E2E 与 M2 收口完成报告

> 日期：2026-07-20
>
> 结论：M2.8.4 已按批准边界完成并通过本地自动化门禁；可重复浏览器 E2E 保留项已关闭。Zalo 宿主真机、生产对象存储/CDN 与越南专业合规复核仍为上线前置条件，本阶段不进入 M3。

## 1. 已交付

- 固定 `@playwright/test@1.61.1`，建立桌面 Chromium、Pixel 7 Chromium 与 iPhone 13 WebKit 三个测试项目；串行执行并在失败时保留 trace 与截图，同时生成 HTML 报告。
- Playwright 同时管理真实 API、管理端、美妆 Mini App 和服装 Mini App 四个本地进程；端口严格独占，不复用配置不明的旧服务。
- 复用既有 M2.6 双商城目录夹具并导出 setup/teardown；全局 setup 创建只具两个测试商城权限的管理员，使用冻结测试 ID 和每轮随机邮箱、密码、TOTP secret，凭据只在进程环境中传递。
- 管理端用例覆盖密码 + TOTP 登录、美妆/服装商城正反哨兵、越南语/中文/英文文案、XLSX 模板下载、真实 API dry-run 上传和 1440×900 无横向溢出。
- Mini App 用例在 Android Chromium 与 iPhone WebKit 分别覆盖两商城名称、行业主题和目录隔离，三语首页、商品详情、SKU 选择、首次 API 失败后的本地化重试恢复，以及移动视口无横向溢出。
- `infra:up` 将长期基础服务健康等待与一次性 `minio-init` 拆分，消除初始化容器成功退出导致的 Compose `--wait` 非零结果，同时保持 bucket 初始化幂等。
- GitHub Actions 安装 Chromium/WebKit、运行独立 E2E 门禁，并在未取消时归档 `playwright-report/` 与 `test-results/`；远程工作流需在后续获授权推送后触发验证。

## 2. 数据、隔离与安全

- 本阶段没有数据库迁移、API 契约或生产环境变量变化；测试只在 `NODE_ENV=test` 下运行已有部署迁移和幂等种子。
- 管理端在美妆商城只观察美妆哨兵并确认服装哨兵不存在，切换服装商城后执行反向断言；Mini App 由两个独立构建时商城配置运行，避免仅依赖前端筛选的假隔离。
- XLSX 浏览器用例只执行 dry-run；正式写入、逐商品事务、跨商城与文件安全边界继续由 M2.8.3 API 集成测试覆盖。
- teardown 后测试管理员、固定商品与页面夹具计数均为 0；凭据未写入文件、报告、浏览器持久化存储或终端输出。

## 3. 验证证据

- `corepack pnpm test:e2e:install`：通过；安装 Chromium 与 WebKit。
- `corepack pnpm infra:up`：通过；PostgreSQL、Redis、MinIO 健康，MinIO bucket 幂等初始化成功。
- `corepack pnpm test:e2e`：连续两轮通过，每轮 3 个项目共 5 项；管理端 1 项，Android Chromium 2 项，iPhone WebKit 2 项，证明同一数据库上的 setup/teardown 可重复。
- `corepack pnpm test:integration`：13 个文件共 59 项全部通过。
- `corepack pnpm verify`：格式、lint、类型、12 个文件共 80 项单元测试、全部生产构建与 Prisma schema 校验通过。
- `corepack pnpm install --frozen-lockfile`、`corepack pnpm audit --prod --audit-level high`、`docker compose config --quiet` 与 `git diff --check`：全部通过；生产依赖无已知漏洞。
- 本机没有安装 gitleaks；针对本阶段运行时邮箱/密码模式的仓库扫描无命中，现有 GitHub Actions gitleaks 步骤保持启用。远程 CI 尚未因本地未推送而执行，不能把 CI 配置更新表述为远程运行通过。

## 4. 已知限制与后续边界

- Web 预览 E2E 不等于 Zalo Mini App 宿主真机验证，不能关闭登录、授权拒绝、手机号 token、手动降级和宿主兼容性验收。
- 本地 MinIO 不代表生产对象存储、CDN、自定义域名、病毒扫描或云密钥管理已验收。
- 可配置的美妆/服装合规门禁不构成越南法律、税务或行业专业意见，上线前仍需专业复核。
- M2 自动化收口完成不授权进入库存、购物车、促销、价格、订单、支付、物流等 M3 领域；下一阶段必须先按 `AGENTS.md` 形成并批准专项计划。
