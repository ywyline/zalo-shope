# M2.6 买家目录 API 与移动端 UI 完成报告

> 日期：2026-07-18
>
> 结论：实现、自动化门禁和真实浏览器验收通过；真实 Zalo Mini App 真机仍按已批准计划结转。

## 1. 交付范围

- 新增公开首页、品牌列表/详情、两级类目树、商品列表和商品详情 API；查询参数使用共享 Zod 契约校验，默认越南语并按资源回退越南语。
- 每个买家请求先从 `X-Store-Code` 解析启用商城，再进入带 `store_id` 的 RLS 事务；只读取启用品牌/类目、已发布商品、启用 SKU 和当前已发布首页版本。
- 首页按模块状态和展示时间窗过滤；内部跳转只返回同商城业务编码，外跳再次执行 HTTPS 主机白名单校验。
- 商品列表支持品牌、根/二级类目过滤和 `newest/price_asc/price_desc` 白名单排序；结构化游标绑定排序类型。商品详情返回本地化规格标签、整数 VND、属性和媒体，但不返回成本价、库存或内部业务 UUID。
- 对象存储适配器新增短期签名读取 URL；READY 媒体响应包含本地化替代文本、宽高和过期时间。基础设施测试实际上传并通过签名 URL 下载校验内容。
- Mini App 新增首页、分类、品牌馆、商品列表、商品详情、SKU 选择、购物车/订单阶段提示和会员页；保留 M1 Zalo 身份与手机号降级流程。
- 美妆与服装商城共用一个路由和组件树，通过商城主题与行业配置呈现差异；新增越南语、中文、英文统一文案、整数 VND 格式、懒加载图片、失败占位、加载/空/错误/重试状态和键盘焦点样式。
- 新增仅用于 local/test 的可重复浏览器数据夹具；它创建两商城可识别的发布内容与真实 MinIO SVG 对象，验收后可完整清理，不进入生产种子。

## 2. 安全与隔离证据

- 集成测试覆盖美妆/服装相同业务编码隔离、错误商城返回 404、未发布/未到展示时间内容不可见，以及不存在商城不泄露资源。
- 品牌、类目、商品、SKU、页面模块和媒体查询均同时约束当前 `store_id`；公开服务仍通过强制 RLS 事务执行，不依赖前端过滤。
- `recommended=false` 不会被字符串真值错误转换；非法语言、页大小、业务编码和排序不匹配游标均返回输入错误。
- 价格从启用 SKU 的 `bigint` VND 转换前检查 JavaScript 安全整数范围；买家 DTO 和回归断言确认不泄露 `cost_price_vnd`。
- 媒体只为 READY 对象生成五分钟读取 URL；签名读取能力保持在存储适配器内，不把供应商实现散落到目录领域。

## 4. 兼容与回滚

- 本阶段不新增数据库迁移，复用 M2.2 已部署模型、RLS 和不可变版本；API 只向 `/v1/catalog` 增加公开读取路径。
- `MediaStorageProvider.createReadUrl` 是存储适配器的向前接口扩展；local/test S3 实现和全部测试替身已同步。应用回滚时数据库无需处理，已发布内容保持不变。
- Mini App 的购物车、库存和下单按钮保持禁用并明确标注 M3；回滚前端不会改变商品、页面或媒体数据。

## 5. 已知限制与后续事项

- 尚未具备真实 Zalo Mini App 项目、测试成员权限和服务端生产 provider 配置，因此 Zalo 宿主内的登录、手机号授权和真实设备尺寸仍未补验；普通浏览器只验证明确的手动降级路径。
- 仓库尚无可重复运行的浏览器 E2E 项目脚本；本阶段提供可重复数据夹具并保留真实浏览器验收证据，但不把交互式验收冒充自动化 E2E。
- local/test 已验证 MinIO 签名上传与读取；生产 CDN、对象存储域名、缓存策略和图片派生尺寸仍需随部署配置落地。
- M2.6 不实现全文搜索、实时库存、计价、购物车或下单。下一步严格按已批准计划进入 M2.7 导入、版本与阶段收口，不自动进入 M3。
