# M2.8.1 商品属性编辑闭环完成报告

> 日期：2026-07-19
>
> 结论：M2.8.1 已完成并通过自动化门禁；只关闭商品属性值编辑 API 保留，不代表 M2 总体验收无保留，也不自动进入 M2.8.2 或 M3。

## 1. 交付结果

- 新增 `GET /v1/admin/catalog/products/{productId}/attributes`，以 `store.catalog.read` 返回商品版本、可编辑状态、固定模板的非规格定义、启用选项和当前类型化属性值。
- 新增 `PUT /v1/admin/catalog/products/{productId}/attributes`，以 `store.catalog.manage`、`expected_version` 和严格判别联合全量替换 `TEXT`、`INTEGER`、`DECIMAL`、`BOOLEAN`、`DATE`、`OPTION` 值。
- 替换仅允许 `DRAFT`/`UNPUBLISHED`；定义必须属于商品固定模板，规格定义继续由 SKU 维护，选项必须属于定义且启用，单值/多值规则由服务端复核。
- 删除旧值、创建新值、条件递增商品版本和 `catalog.product.attributes_replaced` 前后快照审计在同一商城事务内完成；冲突整体回滚。
- 发布门禁只认可当前模板、类型正确的非规格值和启用 SKU 规格；必填文本属性必须存在非空越南语值。
- 修复公共审计 JSON 序列化对 Prisma Decimal 的处理，使用对象的 JSON 标量表示，避免把实现字段/函数写入审计或发布快照。

## 2. 契约、数据与安全

- 请求对象严格拒绝未知字段和 `store_id`；商城仅来自已认证的 `X-Store-Code` 与服务端商城上下文。
- 整数限制为安全整数；定点小数只接受最多 16 位整数、8 位小数的字符串；日期必须是可往返的真实 `YYYY-MM-DD`。
- `TEXT` 明确携带 `vi`/`zh`/`en`。单值文本每种语言最多一项；其他单值类型最多一项；多值拒绝完全重复值。
- 所有查询与写入带商城条件并运行在现有强制 RLS 事务中；跨商城路径 ID 返回 404，读权限不能写。
- 复用现有 `product_attribute_values`、复合外键、one-value check 和商品版本，没有 schema 迁移。数据字典已纠正原先“数据库检查 data_type/部分唯一索引”的超前描述，明确当前由唯一受支持 API 执行跨表与多值校验。

## 3. 验证证据

- `corepack pnpm verify`：通过；格式、lint、类型检查、11 个单元测试文件共 65 项、全部生产构建和 Prisma schema 校验均成功。
- `corepack pnpm test:integration`：通过；11 个文件共 51 项。新增 4 项覆盖读取投影、读写 RBAC、跨商城 404、六种值类型、规格/类型/停用选项拒绝、单值/多值、乐观锁回滚、审计、越南语门禁和非可编辑状态。
- `corepack pnpm audit --prod --audit-level high`：无已知漏洞。
- `docker compose config --quiet`、`git diff --check`、凭据特征扫描：通过。
- 测试清理后 M2.8.1 商品、模板、品牌、类目、管理员和角色前缀计数均为 0，无测试业务残留。

## 4. 兼容与回滚

- 新接口是 `/v1` 向后兼容新增；未修改现有请求或数据库结构。
- 应用可回滚到 M2.7，已保存属性值仍受现有 schema 与 RLS 保护；误修改通过下一次受审替换向前修复，不删除审计或发布版本。
- Decimal 审计修复只把具有 `toJSON()` 的内部标量按其 JSON 表示递归脱敏，不改变普通对象、日期或 bigint 的既有行为，并有回归单测。

## 5. 已知限制与后续事项

- 属性模板 `validation_rules` 尚未冻结版本化结构；本阶段不对任意 JSON 静默执行范围或正则规则。后续引入前必须先冻结契约和兼容策略。
- 品牌/类目/商品/合规管理端 UI、XLSX/商品导出和可重复浏览器 E2E 仍分别属于 M2.8.2-M2.8.4，未在本阶段实现。
- 本阶段没有 UI 改动，因此未新增浏览器验收；这不能关闭 M2 总结中的自动化浏览器 E2E 保留。
- Zalo 真机、生产对象存储/CDN 和越南行业专业合规复核仍依赖外部环境或人员，不能以本地实现替代。
