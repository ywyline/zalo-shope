# M4 地址、结算、订单与 COD 权限矩阵

> 状态：M4 已实施
>
> 日期：2026-07-23
>
> 商城范围：管理员权限必须同时通过访问令牌、目标商城授权、`X-Store-Code`、`store_id` 查询参数与数据库 RLS；会员令牌商城必须等于 Header 商城。

## 1. 新增权限

| 权限 code               | 用途                                             | 风险等级 |
| ----------------------- | ------------------------------------------------ | -------- |
| `store.orders.read`     | 查看目标商城订单列表、详情、脱敏地址快照与时间线 | 高       |
| `store.orders.manage`   | COD 确认、取消、关闭、备注和标签                 | 极高     |
| `store.delivery.read`   | 查看目标商城配送/COD 策略                        | 中       |
| `store.delivery.manage` | 修改配送费、偏远地区、免邮与 COD 阈值            | 极高     |

迁移只把四项 code 登记到权限目录，不给任何生产角色自动授权。local/test 种子显式把它们加入系统 `store-admin`，生产必须通过受审授权流程分配。

## 2. 管理端动作矩阵

| 动作/接口                         | orders read | orders manage | delivery read | delivery manage | 额外控制                                         |
| --------------------------------- | :---------: | :-----------: | :-----------: | :-------------: | ------------------------------------------------ |
| `GET /v1/admin/orders`            |      ✓      |               |               |                 | 逐商城 RLS、状态过滤、游标上限                   |
| `GET /v1/admin/orders/{id}`       |      ✓      |               |               |                 | 地址快照读取单独写审计，手机号掩码               |
| `POST .../confirm-cod`            |             |       ✓       |               |                 | 行锁、状态机、库存消费与审计同事务；重复不再消费 |
| `POST .../cancel`                 |             |       ✓       |               |                 | 待确认 RELEASE；待履约 RESTORE；状态与库存同事务 |
| `POST .../close`                  |             |       ✓       |               |                 | 只允许待确认/待支付，释放活动预留                |
| `PATCH .../notes`                 |             |       ✓       |               |                 | 长度/标签数量限制，前后快照审计                  |
| `GET /v1/admin/delivery-policy`   |             |               |       ✓       |                 | 逐商城 RLS                                       |
| `PATCH /v1/admin/delivery-policy` |             |               |               |        ✓        | `expected_version` 乐观锁、金额安全整数、审计    |

`read` 不隐含 `manage`，前端隐藏按钮不替代服务端判权。管理员 Header 的商城 code 和查询参数 `store_id` 必须共同指向已授权商城；跨商城对象 ID 在当前上下文中按不存在或拒绝处理。

## 3. 买家能力

| 能力                    | 身份                             | 数据边界                             | 敏感数据与一致性控制                                     |
| ----------------------- | -------------------------------- | ------------------------------------ | -------------------------------------------------------- |
| 地址列表/创建/更新/停用 | 商城绑定会员                     | 仅当前会员、当前商城                 | 收货人/手机号/详细地址加密；手机号只返回掩码；更新用版本 |
| 结算报价                | 商城绑定会员                     | 当前会员地址、券和商城商品/库存/策略 | 严格 DTO 不接受价格、store_id 或状态；服务端重算         |
| COD 下单                | 商城绑定会员 + `Idempotency-Key` | 当前会员、商城、地址、购物车、券     | 报价 hash、库存锁、核销、快照和幂等同事务；ONLINE 拒绝   |
| 订单列表/详情           | 商城绑定会员                     | 仅当前会员、当前商城                 | 读取不可变地址快照；手机号掩码                           |
| 买家取消                | 商城绑定会员                     | 仅本人待确认订单                     | 订单行锁、预留 RELEASE、转换同事务；重复幂等             |

会员不能通过更换 `X-Store-Code`、地址 UUID、订单 UUID、游标、SKU code 或优惠券 code 访问其他商城/会员事实。管理端 `store_id` 从不出现在买家请求体。

## 4. 数据库最小权限

- 所有 M4 商城表强制 RLS；运行角色没有跨商城 bypass。
- 订单行、快照和转换撤销 UPDATE/DELETE；历史修正只追加新命令或转换。
- `member_coupons` 的运行角色 UPDATE 被收窄到 `status`、`used_at`、`used_order_id`、`updated_at`，触发器只允许字段完整的一次 `CLAIMED -> USED`。
- 敏感快照密文不能进入 API 错误、结构化日志、审计快照或前端持久存储；后台地址读取必须产生 `order.delivery-address.read` 审计。
- worker 逐商城建立 `StoreContext`，先完成库存预留终态，再关闭相应订单；失败保留计数和错误 code 供重试。

## 5. 稳定拒绝与安全测试

- 未认证：`401 AUTHENTICATION_FAILED`；管理员无商城/权限：`403 AUTHORIZATION_DENIED`；当前主体范围内不存在：`404 RESOURCE_NOT_FOUND`。
- 严格 DTO 或 Header：`400 INPUT_INVALID`；状态、版本、报价、幂等、COD 和库存冲突：`409 CONFLICT` 并按允许集合返回 `details.reason_code`。
- 必测：跨会员地址/订单、令牌/Header 商城不匹配、普通管理员跨商城、只读管理员执行管理命令、客户端金额和状态字段篡改。
- 必测：相同幂等键同/异请求、并发库存不足、过期预留、重复 COD 确认、确认后重复取消、优惠券只核销一次、地址主表修改不影响订单快照。
- 必测：API/日志/审计不泄露密文、完整手机号、Token、SQL、堆栈、内部路径、其他商城 UUID 或请求 hash。
